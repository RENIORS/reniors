# 쿠렌토

WebRTC 미디어 서버 및 클라이언트 API 세트

장점

- 무료
- 자세한 DOCS [https://doc-kurento.readthedocs.io/en/6.9.0/user/about.html](https://doc-kurento.readthedocs.io/en/6.9.0/user/about.html)
- 다양한 클라이언트 API 제공
- 다양한 WebRTC 미디어 서버 유형 지원 ( MCU, SFU 모두 지원 )
    - MCU →
        - 다대다 통신에서 여러 미디어 스트림을 하나의 미디어 스트림으로 통합 후 클라이언트 에게 제공
        - 서버의 CPU 부하가 SFU에 비해 높은편이나, 사용자는 SFU에 비해 더 적은량의 데이터로 미디어 스트림을 받을 수 있음
    - SFU →
        - 다대다 통신에서 미디어 서버는 여러 미디어 스트림을 통합 하기 위한 디코딩, 인코딩 작업 없이 클라이언트에게 제공
        - 서버의 CPU부하가 MCU에 비해 낮은편이나 사용자는 MCU에 비해 더 많은 데이터를 수신
